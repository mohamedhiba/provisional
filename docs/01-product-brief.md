# Product Brief

## Working name

Proof

This name is provisional. It fits the core idea: ambition must become daily proof.

## Mission

Turn ambition into daily proof through ruthless clarity, execution tracking, and accountability.

## Who it is for

Proof is for ambitious, self-directed people who already care deeply about goals like career growth, academics, health, and discipline, but who lose consistency because of drift, overwhelm, emotional noise, and weak accountability.

Primary first user:

- a solo builder or student
- high standards, high ambition
- regularly plans more than they execute
- wants a serious system, not a cute planner

## Main problem

The user does not need more information. The user needs enforced alignment between long-term goals and daily behavior.

The failure pattern is:

- too many priorities
- avoidance disguised as productivity
- weak daily accountability
- no structured review loop
- big goals disconnected from today

## Transformation promised

Proof turns the user from:

- overwhelmed
- reactive
- inconsistent
- self-deceptive

into someone who is:

- clear
- disciplined
- score-driven
- honest
- consistent

## Core daily loop

1. Morning: define the one thing and top three outcomes.
2. Midday: log focus sessions as evidence of real work.
3. Evening: complete a mandatory daily review.
4. Weekly: review patterns, wins, failures, and next adjustments.

## Top 5 MVP features

1. Onboarding that defines mission, pillars, targets, and non-negotiables.
2. Daily command center with one thing, top three, score, and pillar progress.
3. Focus session logger with planned time, actual time, and quality rating.
4. End-of-day review with avoided work, wasted time, and tomorrow's first move.
5. Weekly review with score trends, deep work totals, missed priorities, and next-week focus.

## What it will not include

- AI coaching everywhere
- social features
- mobile app
- browser blocker integrations
- calendar sync
- advanced gamification
- broad note-taking or wiki features
- complex analytics beyond behavior-changing insights

## Success metric after 30 days

The product is successful only if the primary user shows measurable behavioral improvement after 30 days of use:

- higher top-task completion rate
- more weekly deep work hours
- fewer drift days
- higher nightly review completion rate
- visible progress in one real-life target such as applications, study consistency, or project milestones

Primary 30-day benchmark:

- `>= 80%` daily usage
- `>= 70%` nightly review completion
- `>= 20%` improvement in weekly deep work hours from week 1 to week 4

## Tech stack choice

- Frontend: Next.js
- UI: Tailwind CSS + shadcn/ui
- Backend: Next.js server actions and route handlers
- Database: Supabase Postgres
- Auth: Supabase Auth
- Charts: Recharts
- Deployment: Vercel

This stack is chosen for speed, practicality, and portfolio quality. It is enough for MVP without adding infrastructure drag.

## Product principles

1. Behavior first. Product second. Code third. UI fourth.
2. Force prioritization instead of collecting tasks.
3. Make truth visible fast.
4. Reward meaningful execution, not activity theater.
5. Keep friction low for logging and high for avoidance.
6. Build only what strengthens the daily execution loop.

## Tone and feel

The product should feel:

- minimal
- sharp
- serious
- honest
- slightly uncomfortable

It should not feel playful, bloated, or decorative.
